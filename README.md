AMR Cross-Species Co-Resistance Analysis
Overview

This project analyzes antimicrobial resistance (AMR) co-resistance patterns across multiple bacterial species using publicly available NCBI Pathogen Detection data. The goal is to identify shared resistance patterns among E. coli, K. pneumoniae, A. baumannii, and P. aeruginosa.

Data sources:

NCBI Pathogen Detection
BV-BRC

Pipeline Workflow

Run the analysis scripts in the following order:

1. amr_step1.py
Purpose: Loads datasets.

2. ncbi_pathogen_qc_all.py
Purpose: Pivots raw long-format BV-BRC data into a wide-format table.

3. ncbi_merge_and_clean.py
Purpose: Merges pivoted data with NCBI metadata to create:

primary_analysis.csv (E. coli, K. pneumoniae, A. baumannii)
secondary_analysis.csv (P. aeruginosa)

4. ncbi_preprocess.py
Purpose: Standardizes breakpoints (CLSI/EUCAST) and removes expected phenotypes. Also joins metadata using BV-BRC genome API files.

5. ncbi_layer3.py
Purpose: Corrects sampling bias using propensity-based reweighting across species, continent, and breakpoint standard.

6. ncbi_module_detection.py
Purpose: Identifies recurring co-resistance clusters (multi-antibiotic resistance modules) across species using primary_analysis.csv and secondary_analysis.csv.

7. ncbi_full_pipeline.py
Purpose: The ID Fix ‚Äî overwrites synthetic IDs with real BV-BRC Genome IDs and adds isolation source as a covariate. Re-runs Layer 3 with updated covariates.

8. ncbi_temporal_fixed.py
Purpose: Finalizes temporal stability by joining patched IDs with collection years and running temporal analyses.

9. ncbi_enhanced_stats.py
Purpose: Calculates additional trend tests (Cochran-Armitage) and generates the auto-markdown report for meta-analytic ORs.

10. ncbi_validation.py
Purpose: Validates results via 1,000 permutation tests to assess significance of observed ORs
Geographic split analysis (Europe vs Asian/Other isolates)

Notes
All pipeline scripts expect the data/ folder path as ./data/.
Output files will be saved to data/results/.
Ensure date files (*_dates.csv) and genome metadata (*_genome_metadata.csv) are present for accurate analyses.
